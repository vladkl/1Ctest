    При анализе работы исследуемой информационной базы, требуются все или часть показателей производительности. Каждый показатель имеет определенные требования по настройке. Желательно выбирать только те типы показателей, которые реально будут использоваться. Это позволит избежать лишних настроек.

    Оперативные показатели

    Это группа показателей, значения которых доступны в режиме реального времени. Такие показатели нужны для предварительной оценки производительности исследуемой информационной базы.

    Взаимоблокировки

    Этот показатель доступен только, если исследуемая информационная база работает на СУБД Microsoft SQL Server.

    Для возможности получения значений количества взаимоблокировок, необходимо выполнить настройку подключения к Microsoft SQL Server.

    Таймауты

    Этот показатель доступен только, если исследуемая информационная база работает на СУБД Microsoft SQL Server.

    Для возможности получения значений количества таймаутов, необходимо выполнить настройку подключения к Microsoft SQL Server.

    Прочие

    Показатели, значения которых получаются от сервера исследуемой информационной базы. При помощи таких показателей можно оценить:

    • длительность выполнения запросов;
    • длительность ожидания на блокировках;
    • проблемы с параллельностью работы.

    Для СУБД Microsoft SQL Server, работоспособность этих показателей возможна только после назначения роли processadmin пользователю, от имени которого сервер 1С:Предприятия исследуемой базы обращается к серверу СУБД.

    Аналитические показатели

    Группа показателей, предназначенных для сбора и анализа информации об оптимальности работы исследуемой информационной базы. Значения аналитических показателей в режиме реального времени не доступны. Значения таких показателей доступны в режиме просмотра, после выполнения анализа полученных ими данных.

    Анализ запросов

    Позволяет определить наиболее длительные и частые запросы, выполняющиеся в исследуемой информационной базе.

    Анализ ожиданий на блокировках

    Позволяет определить причины ожиданий на блокировках в исследуемой информационной базе.

    Анализ взаимоблокировок

    Позволяет определить причины взаимоблокировок в исследуемой информационной базе и доступен только, если исследуемая информационная база работает на СУБД Microsoft SQL Server 2005 и выше (необходимо выполнить настройку подключения к Microsoft SQL Server).
