Список взаимоблокировок

Список взаимоблокировок в ЦУП представляет собой дерево следующей структуры:

  • Уровень 1. Группы взаимоблокировок;
  • Уровень 2. Взаимоблокировки данной группы;
  • Уровень 3. Последовательность установки блокировок внутри данной взаимоблокировки

На первом уровне взаимоблокировки группируются по совпадению всех ожидающих запросов и их ресурсов. Иначе говоря, группируются взаимоблокировки, которые совпадают по всем своим ожиданиям. Для каждой группы указывается вес и количество взаимоблокировок, которые в нее входят.

На втором уровне находится список взаимоблокировок, которые относятся к данной группе. Для каждой взаимоблокировки указано время ее возникновения.

На третьем уровне показана последовательность установки блокировок в данной взаимоблокировке.

  • Для каждой блокировки отображается следующая информация:
  • порядковый номер процесса, который установил блокировку;
  • полльзователь, выполнивший запрос и его компьютер;
  • момент времени установки блокировки;
  • строка кода конфигурации, из которой был вызван запрос, устано-вивший блокировку;
  • заблокированный ресурс:
    • имя объекта метаданных;
    • имя таблицы;
    • имя индекса;
  • блокировка:
    • тип блокировки;
    • состояние блокировки. Это поле может приобретать следующие значения:
      • Установлена. Блокировка была успешно установлена
      • Ожидание. Блокировка находится в состоянии ожидания
      • Преобразование. Блокировка ожидает снятия конкурирующей блокировки.

При двойном клике по взаимоблокировке будет открыта форма подробной информации о взаимоблокировке.
