Показатели производительности

«Центр управления производительностью» предназначен для сбора и последующего анализа технологических показателей производительности клиент-серверной системы.

ЦУП собирает информацию по трем основным группам показателей:

  • запросы;
  • ожидания на блокировках;
  • взаимоблокировки.

Поскольку в системе, как правило, работает множество пользователей, значения показателей необходимо агрегировать. Например, в данный момент времени происходит выполнение пяти разных запросов от пяти разных пользователей. ЦУП позволяет показывать следующие агрегатные значения большинства показателей:

  • Сумма. Например, суммарное время выполнения всех запросов выполняющихся в данный момент.
  • Максимум. Например, максимальное время выполнения запроса из всех, выполняющихся в данный момент.
  • Среднее. Например, среднее время выполнения всех запросов, выполняющихся в данный момент.
  • Количество. Например, количество запросов, которые выполняются в данный момент времени.


Кроме того, при необходимости, с помощью ЦУП можно собрать так называемые аналитические показатели производительности, содержащие подробную техническую информацию о проблемах производительности, имеющихся в системе.

Данные показатели производительности позволяют анализировать наиболее типичные проблемы производительности, возникающие в работающих системах. В дальнейших версиях ЦУП набор показателей производительности может быть расширен.

Запросы

Данная группа показателей предназначена для отображения времени выполнения запросов в системе.

Относительно большая длительность запросов может говорить об их неоптимальной работе. В случае если значения данных показателей производительности указывают на наличие неоптимально работающих запросов, необходимо собрать аналитическую информацию по работу запросов и оптимизировать запросы в соответствии с рекомендациями, данными в настоящем руководстве.

Ожидания на блокировках

Показатели данной группы отражают время ожидания на блокировках при работе пользователей с общими данными. Ожидание на блокировке возникает в том случае, если пользователь не может захватить некоторые данные (например, остатки регистра для записи) из-за того, что данные уже захвачены другим пользователем.

ЦУП отображает значения следующих показателей производительности, связанных с ожиданиями на блокировках:

  • Все блокировки. Суммарное время ожидания на блокировках уровня СУБД и блокировках уровня сервера 1С:Предприятия (для управляемых транзакций)
  • Только блокировки СУБД. Время ожидания на блокировках уровня СУБД.
  • Только блокировки 1С. Время ожидания на блокировках уровня 1С:Предприятия (для управляемых транзакций).

Относительно большое время ожидания на блокировках может говорить о наличии в системе избыточных блокировок, которые снижают производительность и общую пропускную способность системы. В случае если значения данных показателей производительности указывают на наличие избыточных блокировок, необходимо собрать аналитическую информацию по ожиданиям на блокировках и оптимизировать систему в соответствии с рекомендациями, данными в настоящем руководстве.

Взаимоблокировки

Взаимоблокировка – это неразрешимый конфликт блокировок, который свидетельствует о наличии ошибочных решений в коде конфигурации и/или структуре метаданных системы.

Взаимоблокировка – событие, не имеющее длительности. По этой причине смысл показателя «Количество взаимоблокировок» отличается, например, от смысла показателя «Количество запросов». Количество запросов отражает, сколько запросов выполнялось (началось, но не закончилось) на момент получения значения этого показателя. Количество взаимоблокировок отражает, сколько взаимоблокировок произошло с момента предыдущего получения значения показателя до момента текущего получения значения.

По этой же причине группа «Взаимоблокировки» имеет только показатель «количество» и не имеет других агрегатных значений.

Значение показателя «количество взаимоблокировок» отличное от нуля, однозначно свидетельствует о наличии в системе неоптимального кода. В этом случае необходимо собрать аналитическую информацию по взаимоблокировкам и оптимизировать систему в соответствии с рекомендациями, данными в настоящем руководстве.

Аналитические показатели

Эксперт оценивает производительность системы на основании:

  • субъективных ощущений пользователей в случае анализа реально работающей системы;
  • прикладных показателей производительности в случае моделирования нагрузки в системе при помощи Тест-центра;
  • значений технологических показателей производительности, полученных при помощи ЦУП.

Если в результате такой оценки производительность системы признается недостаточной, необходимо предпринять шаги по повышению производительности системы. ЦУП позволяет собрать так называемые аналитические показатели производительности, которые содержат подробную техническую информацию обо всех проблемах производительности, имеющихся в системе. При помощи этой информации можно провести оптимизацию кода системы и структуры метаданных с целью повышения производительности системы.
